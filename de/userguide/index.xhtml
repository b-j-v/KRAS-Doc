Allgemeine Hinweise

Bitte beachtet, dass dieses System auf Internet-Technologie basiert! Das heißt, es kann schon einmal vorkommen, dass der gewünschte Vorgang etwas länger dauert! Es ist dann nicht sinnvoll, die Taste sofort mehrmals zu drücken, da so ungewünschte Effekte entstehen können, wie z.B. die doppelte Anlage von Terminen!

Das System ist durch Benutzername und Passwort gegen Zugriff von außen geschützt! Nur Kampfrichter haben Zugang zu diesem System!

Da dieses System nur registrierten Benutzern Zugang gewährt, werdet ihr automatisch nach einer vorgegebenen Zeit, innerhalb der von euch keine weitere Aktion registriert wird, wieder abgemeldet. Dies dient der Sicherheit, damit keiner Zugriff nehmen kann, falls ihr euch z.B. versehentlich nicht ausgeloggt habt.

Systemvoraussetzungen für die Benutzer

Für die Benutzer gibt es ein paar Punkte zu beachten, um das System optimal nutzen zu können:

KRAS wurde gemäß der Standards des World Wide Web Consortiums (W3C) entwickelt. Als Benutzerprogramm wurde Mozilla Firefox zum Testen eingesetzt. Es wird daher die Nutzung dieses Browsers empfohlen. Allerdings sind auch die Benutzung anderer Browser (z.B. Google Chrome oder Internet Explorer) grundsätzlich möglich. Einschränkungen sind nicht zu erwarten, solange die aktuellen Versionen benutzt werden.
Insbesondere vom Einsatz des Internet Explorer vor Version 8 wird jedoch dringend abgeraten!

Das System arbeitet mit JavaScript-Elementen. Bitte schaltet also für diese Seite JavaScript ein, um den vollen Umfang der Anwendung nutzen zu können. Die Skripte dienen hauptsächlich zur Plausibilitätsprüfung bzw. zur Verhinderung von Fehleingaben, also um euch das Leben etwas leichter zu machen! Ohne JavaScript werden euch diese Arbeitserleichterungen nicht zur Verfügung stehen. Es stehen aber alle Funktionalitäten zur Verfügung.

Die Menüs im Überblick

Eigene Daten anzeigen/ändern

Dieser Menüpunkt dient zum Bearbeiten des eigenen Eintrags in der Kampfrichterkartei.
Ebenso kann hier der Link für den persönlichen Kalender (iCal) abgerufen werden.

KR-Kartei einsehen

In diesem Menüpunkt ist eine Liste der Kampfrichter einsehbar. Das bedeutet, es werden alle Informationen sichtbar. Da die Daten hier vom Nutzer direkt gepflegt werden können, werden die Daten hier immer auf dem neuesten Stand sein! Die Daten der einzelnen Kampfrichter können hier auch als vCard abgerufen werden.

Anmeldung zu Terminen

Dieser Menüpunkt führt zu einer Liste mit den Terminen, die für den angemeldeten Benutzer relevant sind und in der man sich für die gewünschten Veranstaltungen anmelden kann.

Sollten bestimmte Ebenen fehlen, so ist dies dem zuständigen Mitglied der KR-Kommission zu melden, das dann die entsprechenden Rechte zuteilt.

Anzeige eingeteilte Termine

Dieses Menü gibt eine Übersicht über alle Termine aus, für die man vom zuständigen Obmann eingeteilt wurde. Das dient vor allem der Übersicht und zu Zwecken des übersichtlichen Ausdrucks.

Notenarchiv

Über diesen Menüpunkt kann eine Liste der Veranstaltungen und der zugehörigen Note der letzten Jahre abgerufen werden, für die man eingeteilt war.

Wie weit dieses Archiv in die Vergangenheit reicht, kann individuell bestimmt werden.

Die Menüs im Detail

Eigene Daten anzeigen/ändern

Nach Auswahl des Menüpunktes erscheint der KR-Eintrag des angemeldeten Kampfrichters.

Im Formular können nur folgende Felder geändert werden:

  • Adresse, PLZ, Ort
  • Verein
  • Nummer des Mitgliedsausweises
  • eMail-Adresse
  • Telefonnummern
  • Zugangsdaten zum System

Alle anderen Angaben müssen dem zuständigen Mitglied der KR-Kommission mitgeteilt werden, falls sich hier etwas geändert hat!

iCalendar-Anbindung

Aktiviert man die Checkbox iCalendar URL neu berechnen und danach den Button speichern , errechnet das System eine individuelle URL für den persönlichen Einsatzkalender. Diese URL wird daraufhin an dieser Stelle angezeigt.
Sollte bereits eine URL bestanden haben, so wird diese damit ungültig!

Das System bietet zwei URLs an:

  • Die erste URL enthält sämtliche Termine, für die man sich angemeldet hat. Dabei werden Termine, für die man nicht eingeplant ist, als transparent und Termine, für die eine Einladung verschickt wurde, als confirmed markiert.
  • Da die o.g. Markierungen leider nicht von allen Kalender-Anwendungen korrekt umgesetzt werden, können mit dem zweiten Link nur die Termine geladen werden, für die man eingeteilt ist.

Die URL kann in jedem beliebigen Kalender benutzt werden, der das iCalendar-Protokoll unterstützt.

Button Speichern

Mit Klick auf den Button werden die eingegebenen bzw. veränderten Daten erst festgeschrieben. Wechselt man ohne vorheriges Speichern in ein anderes Menü, werden sämtliche Änderungen verworfen!
Beim Speichern wird das Änderungsdatum und der Ändernde abgespeichert.

Bemerkungen zu den Feldern:

eMail:

Bitte beachtet die korrekte Schreibweise der eMail-Adresse! Die hier angegebene eMail-Adresse wird auch für den Versand der Einladungen und Erinnerungen durch das System verwendet!!!
Aus diesem Grund darf dieses Feld auch nicht leer sein! Ist dies der Fall, wird um Eingabe einer gültigen eMail-Adresse gebeten (bei aktiviertem JavaScript). Eine zweite eMail-Adresse kann, falls nötig, durch Komma getrennt angefügt werden.

Die Adressen werden auf syntaktische Korrektheit geprüft, d.h. ob sie in der Schreibweise richtig sind. Ist dies nicht der Fall, wird die entsprechende Adresse durch einen Fehlertext ersetzt und ein Hinweis beim Speichern ausgegeben!

Benutzer: Der Benutzername muss natürlich eindeutig sein. Wird ein Name gewählt, der schon existiert, wird eine entsprechende Fehlermeldung erscheinen und der alte Benutzer bleibt aktiv.
Passwort:

Wenn das Passwort geändert werden soll, muss zur Absicherung gegen Tippfehler das Passwort ein zweites Mal eingegeben werden! Stimmen die beiden Eingaben nicht überein, erfolgt eine Fehlermeldung und das Passwort wird nicht geändert. Alle anderen Änderungen werden aber gespeichert.

Wenn die Passwort-Felder leer sind, wird das Passwort ebenfalls nicht geändert!

KR-Kartei einsehen

Nach Auswahl des Menüpunktes wird eine Liste aller Kampfrichter ausgegeben, die nach den Bezirken gruppiert ist.

Klickt man auf die eMail-Adresse eines bestimmten Kampfrichters, wird ein eMail-Fenster mit seiner eMail-Adresse im Empfängerfeld geöffnet und man kann ihm direkt eine eMail schreiben.
Hinweis: Für diese Funktion muss ein eMail-Client auf dem Rechner installiert sein!

Durch Klick auf den Namen eines Kampfrichters, wird dessen Karteieintrag angezeigt.

Button Nach Name sortieren

Durch Klick auf den Button werden alle Kampfrichter ohne Gruppierung dem Alphabet nach geordnet.

Button nach Bezirk sortieren

Durch Klick auf den Button werden die Kampfrichter nach den Bezirken gruppiert angezeigt.

Anmeldung zu Terminen

Das Menü zeigt eine Liste aller Veranstaltungen, für die man sich anmelden kann. Welche Veranstaltungen das sind, hängt von der jeweiligen Ebene ab und wird durch das zuständige Mitglied der KR-Kommission eingestellt. Sollte also eine Gruppe von Veran-staltungen fehlen, muss die Berechtigung zur Anmeldung in der entsprechenden Ebene von diesem vorgenommen werden.

Hinweis: Die Zuordnung zu den Ebenen können unter Eigene Daten anzeigen/ändern überprüft werden!

Neue bzw. veränderte Termine (seit der letzten Anmeldung) und solche, für die bereits eine Einteilung verschickt wurde, werden gesondert hervorgehoben (standarmäßig fett bzw. grün).

In der ersten Spalte befindet sich vor jedem Eintrag eine Checkbox zur Auswahl der gewünschten Termine. Hier müssen alle Termine angeklickt werden, für die man sich melden möchte.
Es ist auch kein Problem, sich für Termine zu melden, die sich überschneiden, da die Einteiler dann eine Meldung bekommen, dass man schon anderweitig eingeplant ist und sich so abstimmen können.

Für den Fall der Verhinderung, können die Häkchen auch entfernt werden, solange man vom zuständigen Obmann noch nicht nominiert wurde.
Ist man bereits für den Termin nominiert, erscheint das Häkchen grau und lässt sich nicht mehr entfernen!
In diesem Fall ist eine Rücksprache mit dem jeweiligen Obmann erforderlich!!!
Ob der jeweilige Obmann die Absage akzeptiert und hinterlegt hat, lässt sich im Menü Anzeige eingeteilte Termine feststellen – dort erscheint der betreffende Termin durch-gestrichen.

Die Häkchen werden erst durch Anklicken des Buttons Speichern festgeschrieben. Wird vorher in ein anderes Menü gewechselt werden alle Veränderungen verworfen!

Beim Speichern wird das Datum der Eintragung hinterlegt.

Button nach Datum sortieren

Durch Klick auf den Button wird die Liste ohne Gruppierung dem Datum nach geordnet.
Lehrgangstermine bleiben davon unberührt.

Button nach Ebene sortieren

Durch Klick auf den Button werden die Veranstaltungen nach der jeweils zugehörigen Ebene bzw. Bezirk gruppiert angezeigt.
Lehrgänge werden hierbei als eigene Gruppe dargestellt.

Anzeige eingeteilte Termine

In diesem Menü wird eine Liste aller Termine ausgegeben, für die man nominiert wurde.
Termine, die durchgestrichen (änderbar!) dargestellt werden, sind Termine, für die der Benutzer nominiert war, abgesagt hat und der zuständige Obmann hat die Absage bereits eingepflegt.
Veränderte Termine (seit der letzten Anmeldung) und solche, für die bereits eine Einteilung verschickt wurde, werden gesondert hervorgehoben (standardmäßig fett bzw. grün).

Die Liste lässt sich mit der Druckfunktion des eigenen Browsers auch ausdrucken.

Mit Hilfe des Radiobuttons vor jeder Zeile kann die jeweilige Veranstaltung ausgewählt werden und mit Klick auf „Absage“ eine eMail an den zuständigen KRO geschickt werden. Dazu im darauffolgenden Fenster einen oder mehrere KROs auswählen, eine Begründung schreiben und auf abschicken klicken.
Die eMail wird dem Sender zur Kontrolle auch selbst zugestellt. Auf Wunsch kann eine Empfangsbestätigung angefordert werden (durch Klick auf die entsprechende Checkbox).

Mit den Links „FG“ und „Einteilung“ kann die Fahrgemeinschaft, für die man eingeteilt ist bzw. die gesamte Einteilung betrachtet werden. Diese öffnen sich in einem neuen Fenster. Mit einem Klick auf den Namen des jeweiligen Kampfrichters kann man direkt seinen Karteieintrag ansehen.

Notenarchiv

Hier werden Veranstaltungen der Vergangenheit mit der hierfür erhaltenen Note angezeigt. Wie weit dieses Archiv zurückreicht, bestimmt die KR-Kommission.

Das Archiv dient der Übersicht und der eigenen Leistungskontrolle!
Deshalb wird am Ende der Liste die Durchschnittsnote des aktuellen und des Vorjahres angezeigt. Es gehen die Lehrgangsnote des betreffenden Jahres und alle Bewertungen vom 01.11. des Vorjahres bis zum 31.10. des lfd. Jahres in die Berechnung ein. Diese Berechnungsgrenze ist im System einstellbar.

Legende

Da wie bereits erwähnt die verschiedenen Formatierungen (z.B. fett, grün oder durchgestrichen) angepasst werden können, wurde an dieser Stelle eine Übersicht aller vorhandenen Spezialformatierungen eingefügt. Mit dieser Übersicht kann jeder nachvollziehen, was welches Format zu bedeuten hat.

Bitte beachten: Die verschiedenen Formate können sich auch überschneiden (wenn z.B. ein Termin abgesagt ist, der gleichzeitig seit dem letzten Besuch aktualisiert wurde).

Downloadarea

Dieser Menüeintrag ist ein Link auf eine Seite, die vom Administrator frei gewählt werden kann. Auf dieser Seite sollen dann Formulare, Meldungen, Ausschrei-bungen u.ä. heruntergeladen werden können.

Durch einen Klick auf diesen Eintrag wird ein neues Fenster mit der gewählten Adresse geöffnet.

Logout

Mit Klick auf den Button Logout wird der Benutzer vom System abgemeldet. Es sollte immer der Logout-Button benutzt werden, da so ein Missbrauch erschwert werden kann.

Meldet sich der User nicht ab, wird er automatisch nach einer vorgegebenen Zeitspanne vom System als abgemeldet gespeichert und beim nächsten Login daran erinnert, sich selber abzumelden.

Beim Abmelden wird auch das Datum hinterlegt, wann der Benutzer zuletzt online war. Hatte er sich nicht abgemeldet, wird dies bei der nächsten Anmeldung geschehen.

Glossar

  • Administrator

    die Person, die das Kampfrichter-Anmelde-System (KRAS) verwaltet bzw. weitgehend alle Rechte auf die Datenbank hat, um Daten zu ergänzen, zu pflegen, etc.

  • alphanumerisch

    In Datenbanken und elektronischen Formularen kann grob in zwei Arten von Feldern unterschieden werden:

    In die einen Felder können nur Zahlen geschrieben werden – ob diese Zahlen ganze Zahlen sind oder Kommazahlen ist dabei unerheblich.
    Diese Felder sind numerisch.

    In die anderen Felder können sowohl Zahlen als auch Buchstaben und z.T. auch Satz- und Sonderzeichen geschrieben werden.
    Diese Felder sind dann alphanumerisch.

  • BJV

    Bayerischer Judo-Verband e.V.

  • (Web-)Browser

    KRAS ist ein webbasiertes System, d.h. um es nutzen und damit arbeiten zu können, muss ein sogenannter Internet-Browser auf dem lokalen Rechner (mit Internetzugang) installiert sein. (s. Systemvoraussetzungen für die Benutzer )

  • Checkbox

    Eine Checkbox ist eine kleine quadratische Fläche, die innerhalb elektronischer Formulare benutzt wird, um bestimmte Felder zu markieren. Eine Checkbox kann angeklickt – dann beinhaltet sie entweder ein Kreuz oder ein Häkchen – oder nicht angeklickt sein – dann ist sie leer.
    Außerdem kann es sein, dass eine Checkbox deaktiviert ist – sie ist dann grau hinterlegt. Das bedeutet, dass sie im Moment nicht verändert werden kann und dient der Information.

  • CSV-Datei

    CSV steht für C omma S eparated V alues

    Eine CSV-Datei ist die einfachste Form der Speicherung von Daten in Datensätzen, wobei ein Datensatz eine Reihe von Daten darstellt. Jeder Datensatz ist inhaltlich und formal gleich aufgebaut innerhalb einer solchen Datei.

    Die einzelnen Daten werden in einer CSV-Datei durch ein Semikolon („;“) getrennt und ein Datensatz ist einfach die Menge aller durch Semikolon getrennten Daten in einer Zeile.

    Es liegt also eine Art primitive Tabelle vor und tatsächlich sind die meisten der Tabellenkalkulationsprogramme in der Lage, ihre Tabellen als CSV-Dateien abzulegen. Solche Dateien können also zum Datenaustausch von sonst inkompatiblen – also nicht zusammenarbeitenden – Programmen herangezogen werden.

  • Default

    Grund- bzw. Voreinstellungen

  • Datenbank

    Die Datenorganisation und Datenhaltung (siehe MySQL )

  • Drop-Down-Feld

    Ein Drop-Down-Feld dient der Dateneingabe innerhalb elektronischer Formulare. Solche Felder begrenzen die Eingabewerte auf eine Liste von Möglichkeiten, die durch das Aufklappen (engl. (to) drop down) sichtbar gemacht werden können. Das Aufklappen erfolgt durch das Anklicken des Pfeils bzw. Dreiecks am Ende des Feldes, dass mit der Spitze nach unten zeigt.

  • Internetprovider

    Die Person bzw. Firma, die den Dienst „Internet“ zur Verfügung stellt. Im Zusammenhang mit KRAS ist die Person/Firma gemeint, die einen Server mit (Web-)Zugang zum Internet bereitstellt.

  • Kampfrichterkommission

    Leitungsgremium der Kampfrichter – es besteht aus dem Landeskampfrichterreferenten und dessen Stellvertreter, den Gebietskampfrichterobleuten Nord und Süd und dem beratenden Mitglied.

  • Kampfrichterobmann bzw. Kampfrichterobleute

    Funktionär(e) im Bayerischen Judoverband e.V. - Leiter einer Einheit auf einer Ebene.

  • (Landes-)Kampfrichterreferent

    Funktionär im Bayerischen Judoverband e.V. - Leiter des Kampfrichterressorts im Verband.

  • KR

    Kampfrichter

  • KRAS

    steht für K ampf r ichter- A nmelde s ystem

  • KRO

    steht für K ampf r ichter o bmann

  • LKRR bzw. KRR

    steht für L andes- K ampf r ichte r referent

  • Lizenz

    Lizenzen der Kampfrichter! Nicht zu verwechseln mit Softwarelizenzen.

  • Memofeld

    Memofelder dienen der Eingabe von längeren Informationen und ganzen Texten in einer Datenbank oder elektronischem Formular.

    Innerhalb von Memofeldern ist es sogar möglich, Zeilensprünge zu machen, um den Text zu strukturieren.

  • MySQL(-Server)

    Eine im Internet frei verfügbare Datenbank, auf die auch unser System zurückgreift.
    Auf dem Webserver sollte mindestens Version 4.1. installiert sein.

  • PDF

    Dokumentenformat zum Austausch von Dokumenten über das Internet. (Quasistandard)

  • PHP

    Eine im Internet frei verfügbare Skriptsprache, auf die auch unser System zurückgreift.
    Auf dem Webserver sollte mindestens Version 4.3.10 installiert sein.

  • phpMyAdmin

    Frei verfügbares Programm bzw. PHP-Skriptsammlung zur komfortablen Pflege einer MySQL -Datenbank. Dieses Programm sollte auf dem Web-Server installiert und nur dem Administrator zugänglich sein.

  • Query

    siehe SQL-Statement

  • Radiobutton

    Ein Radiobutton allein existiert nicht – es sind immer mindestens zwei. Radiobuttons dienen der Auswahl von sich ausschließenden Alternativen – man kann also immer nur eine der möglichen Vorgaben auswählen. Klickt man eine andere Vorgabe an, so verschwindet die vorige Auswahl.
    Radiobuttons sind als runde kleine Kreise dargestellt und der ausgewählte Zustand wird dadurch markiert, dass der Kreis gefüllt ist.

    Radiobuttons können auch deaktiviert sein und sind dann grau hinterlegt. Das bedeutet, dass die angegebene Auswahlmöglichkeit aus bestimmten Gründen nicht wählbar ist.

    Der Begriff ist entstanden, weil diese Steuerelemente ähnlich wie bei alten Radios funktionieren. Dort konnte man ein Frequenzband auswählen, indem man auf den dazugehörigen Knopf drückte. Dadurch sprang aber automatisch der Knopf des Bandes, das zuvor gewählt war, wieder heraus.

  • Screenshot

    Abbildung eines Bildschirminhalts bzw. Fensters von KRAS.

  • SQL-Statement

    Diejenigen Anweisungen für die Datenbank, um bestehende Daten ändern, ergänzen, löschen oder abrufen zu können.

  • Web

    world wide web – ist das, was in der Regel unter Internet verstanden wird. Um Web-Seiten lesen zu können, wird ein Browser benötigt.

  • Webserver

    Ein Rechner, der Dienste wie das Web und eMail zur Verfügung stellt.
    Welche Software bzw. welches Betriebssystem (z.B. Linux, Unix, Windows XP-Server) hierfür eingesetzt wird, ist für KRAS unerheblich.
    Für das KRAS wird noch verschiedene weitere auf dem Server installierte Software benötigt: PHP, mySQL.
    Die benötigte Hardware (Prozessorleistung, Speicher etc.) richtet sich nach der eingesetzten Software (vgl. Handbuch PHP, mySQL etc.)

  • Webspace

    physikalischer Speicherplatz auf einer Festplatte, die in einem Webserver eingebaut ist.
    KRAS benötigt minimal ca. 3 MB. Zusätzlich wird, abhängig von der Anzahl der Datensätze, entsprechend Platz in der Datenbank benötigt.
